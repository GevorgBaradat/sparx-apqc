## 2. Главный экспортёр SVG

### `ea_diagram_to_svg_v38.js`

### 2.1 Что делает (для разработчика)

**Источник данных:** SQLite-база модели EA (`.qea`/`.xea`), читается через `Repository.SQLQuery()`.

**Что читается:**

- `t_diagramobjects` + `t_object` → элементы диаграммы, их позиции (RectLeft/Top/Right/Bottom), тип, стереотип, заметки (`o.Note`), признак drill-down
- `t_diagramlinks` + `t_connector` → коннекторы, стиль (`Style`), геометрия (`Geometry`), waypoints (`Path`), цвет
- `t_diagram.ParentID` — class drill-down: элемент → дочерняя диаграмма
- `NType=8` + `PDATA1` — activity/BPMN drill-down

**Система координат:** EA использует Y-вниз с отрицательными значениями.  
Преобразование: `svgX = eaX − eaMinX + PAD`, `svgY = eaMaxY − eaY + PAD` (инверсия Y).

**Геометрия линий — три пути:**

| Признак | Стиль | Алгоритм |
|---|---|---|
| Нет `Path` и нет `TREE=` | **Direct** | якорь = centre(el) + (SX, −SY); клип отрезка к боксам (Liang–Barsky) |
| Есть `Path`, есть `TREE=OS` | **Ortho-Square** | точки из `dl.Path` + граневая привязка по `EDGE`+`SX/SY` |
| Есть `TREE=V/H/LV/LH` | **Tree** | 4-точечный маршрут через горизонтальный/вертикальный hub |

**Маркеры коннекторов:** `SourceIsAggregate`/`DestIsAggregate` (1=пустой ромб, 2=закрашенный).  
Fallback (нет явных флагов): ромб на END стороне (`meId="dia-e"`), стрелка навигации подавляется автоматически.

**Drill-down (два механизма):**
1. **Activity/BPMN:** `NType=8 AND PDATA1 != 0` → `PDATA1` = ID дочерней диаграммы
2. **Class:** `t_diagram.ParentID = element.Object_ID` → `MIN(Diagram_ID)` по GROUP BY

**Рекурсивный обход дерева:** `VISITED[diagID]` защищает от циклов/повторов; имена файлов — `ИмяДиаграммы_ID.svg` (уникальны); ссылки — относительные, все файлы в одной папке.

**SVG-структура:**
```
<svg>
  <defs>  <!-- маркеры стрелок/ромбов -->
  <style> <!-- классы элементов -->
  <rect id="ea-bg">
  <text>  <!-- название диаграммы -->
  <g id="ea-scene">   <!-- весь контент диаграммы (масштабируется отдельно) -->
    <!-- коннекторы (<path data-orig-d="...">) -->
    <!-- элементы (<g class="ea-el" data-id="..." data-name="...">) -->
  </g>
  <g id="ea-tip">     <!-- тултип (SVG-слой) -->
  <foreignObject id="ea-ui">  <!-- HTML-оверлей: панель, модальное окно -->
  <script><![CDATA[
    var EA_NOTES = {...};   // Notes элементов
    var EA_RECTS = {...};   // SVG-позиции и размеры элементов
    (function(){ ... })();  // браузерный IIFE
  ]]></script>
</svg>
```

**Браузерный IIFE делает:**
- `layout()` — синхронизирует размер SVG с зумом и окном браузера
- тултипы: `mouseover/mousemove/mouseout` на `.ea-el`
- окно замечаний: `openEditor/saveEditor/closeEditor`, re-seed Notes при каждом открытии
- panель: drag по заголовку, resize за угловой grip
- fisheye: `applyFish/resetFish`, покоординатная трансформация путей и `transform` на группах
- сохранение: `XMLSerializer` + `showSaveFilePicker` / fallback download
- восстановление комментариев: `restore()` при загрузке по `data-comment`

### 2.2 Как пользователь видит SVG-файл

SVG открывается в браузере (Chrome, Edge, Firefox) как полноценное интерактивное приложение.  
Скрипт открывает файл автоматически после выгрузки.

**Что видно на экране:**
- Диаграмма EA — элементы (блоки с именем и стереотипом), линии с маркерами
- На элементах с дочерними диаграммами — маленький значок 🔗 в углу (две скрепки)
- На элементах с сохранёнными замечаниями читателя — оранжевый треугольник в правом верхнем углу
- Панель масштабов в правом верхнем углу (перемещаемая, изменяемая)

---

### 2.3 Панель управления

Панель находится в правом верхнем углу. Её можно:
- **Перетащить** за заголовок «МАСШТАБ» в любое место экрана
- **Изменить размер** — потянуть за треугольный grip в правом нижнем углу

**Секция МАСШТАБ — три независимых ручки:**

| Ручка | Что делает |
|---|---|
| Диаграмма +/− | Масштабирует содержимое SVG; холст растёт вслед за содержимым → появляется прокрутка |
| Тултип +/− | Масштабирует всплывающую подсказку |
| Окно +/− | Масштабирует окно редактирования замечаний |

> Тултип и окно всегда в экранном масштабе — не зависят от зума диаграммы.  
> При зуме диаграммы они не «уплывают» и не обрезаются.

**Кнопка «Сохранить с заметками…»** — сохранить SVG с текущими замечаниями в новый файл.

**Секция FISHEYE:**

| Элемент | Что делает |
|---|---|
| Кнопка 👁 вкл/🐟 выкл | Включить/выключить линзу |
| Сила D +/− | Интенсивность увеличения в центре линзы (1–15) |
| Радиус R +/− | Размер зоны линзы в пикселях (50–400) |

### 2.4 Тултипы (всплывающие подсказки)

**Появляются:** при наведении курсора на элемент, у которого заполнено поле **Notes** в EA.

**Содержимое:** текст поля Notes элемента модели, с переносом строк.

**Поведение:**
- Следует за курсором мыши
- Автоматически уходит от края экрана (не обрезается)
- Масштабируется регулятором «Тултип» в панели
- Исчезает при уводе курсора за пределы элемента

**Курсор** над элементом с заметкой меняется на `pointer`.

> Тултип — **только для чтения**. Редактирование — через клик (см. §2.5).

---

### 2.5 Окно редактирования замечаний

**Открывается:** кликом по любому элементу на диаграмме.

**Что видит пользователь:**
- Заголовок: «Примечания: [имя элемента]»
- Текстовое поле с содержимым: **текст Notes** из модели сверху, затем горизонтальная черта `──────────`, затем ранее сохранённые замечания (если были)
- Кнопки **«Сохранить»** и **«Закрыть»**

**Логика текстового поля:**  
При каждом открытии текст Notes **вставляется заново сверху** как заготовка-шаблон.  
Пользователь может:
- Удалить весь вставленный блок Notes (он не нужен — только мешает)
- Взять из него фрагмент и переформулировать
- Написать своё замечание под чертой
- Нажать «Сохранить» — замечание сохраняется **в SVG-файле**

**После сохранения:** в правом верхнем углу элемента появляется **оранжевый треугольник** — маркер того, что по этому элементу есть замечание читателя.

**Окно можно перетащить** за синий заголовок в любое место экрана.

**Масштаб окна** настраивается регулятором «Окно» в панели.  
При открытии окно центрируется и автоматически уменьшается, если не вмещается в экран.

### 2.6 Навигация по диаграммам

#### Переход вглубь (drill-down)

Элементы, у которых есть дочерняя диаграмма детализации, отмечены значком **🔗** (две скрепки) в правом нижнем углу.

**Действие:** клик на значок → браузер открывает дочернюю диаграмму в том же окне.

Поддерживаются два типа связей:
- **Activity / BPMN**: дочерняя диаграмма задана через `NType=8 + PDATA1`
- **Class**: дочерняя диаграмма найдена через `t_diagram.ParentID = element.OID`

#### Возврат назад (drill-up)

**Кнопка «Назад» браузера (←)** — возвращает на предыдущую диаграмму.  
**Кнопка «Вперёд» браузера (→)** — снова переходит вниз по цепочке.

> Навигация работает как обычные гиперссылки между HTML-страницами.  
> Все файлы дерева лежат **рядом в одной папке** — относительные ссылки работают  
> без веб-сервера, просто из папки на диске.

#### Что происходит при экспорте

При запуске скрипта с открытой стартовой диаграммой автоматически экспортируется **всё дерево** дочерних диаграмм рекурсивно до листьев. В Session Output выводится:
```
[build] Yuriy Chernyavskiy (ID=10015) -> Yuriy_Chernyavskiy_10015.svg
  [build] Enrico Tebaldi (ID=9823) -> Enrico_Tebaldi_9823.svg
  [build] Academic / Researcher (ID=10016) -> Academic___Researcher_10016.svg
=== Экспортировано диаграмм дерева: 3 ===
```

**Защита от циклов:** карта `VISITED[diagID]` — каждая диаграмма экспортируется один раз. Ромбовидные/циклические графы (A→B, A→C, B→D, C→D) обрабатываются корректно.

### 2.7 Масштабирование и прокрутка

**Кнопки «Диаграмма +/−»** в панели:
- При увеличении холст SVG растёт → браузер показывает полосы прокрутки
- Содержимое никогда не обрезается
- При уменьшении холст не меньше размера окна браузера

**При resize окна браузера** — пересчёт `layout()` автоматически.

---

### 2.8 Fisheye в SVG (v38)

Когда fisheye включён кнопкой в панели:

- Курсор над диаграммой меняется на перекрестие
- При движении мыши по `#ea-scene` вокруг курсора создаётся линза
- Элементы вблизи курсора увеличиваются, дальние сжимаются, но остаются видимы
- Синее пунктирное кольцо показывает границу линзы

**Технически:** элементы трансформируются через `translate(fx,fy) scale(s) translate(-cx,-cy)` на группе `<g>`. Коннекторы пересчитываются покоординатно из `data-orig-d` (поэтому все коннекторы в v38 сохраняются как `<path>`, а не `<line>`).

При зуме диаграммы (diagScale) координаты мыши делятся на diagScale для правильной привязки в системе координат сцены.

---

### 2.9 Сохранение с замечаниями

**Кнопка «Сохранить с заметками…»:**
- Сериализует весь DOM текущего SVG (включая замечания в `data-comment` и оранжевые метки)
- Предлагает выбрать имя и папку нового файла (File System Access API)
- В браузерах без File System Access API — стандартная загрузка файла

**При повторном открытии** сохранённого файла — все замечания и метки восстанавливаются автоматически (`restore()` читает `data-comment` из DOM).

## 3. Fisheye-просмотрщики

> **⚠️ Статус: дослідження / прототипування**  
> Технологія «рибячого ока» — на стадії дослідження.  
> **Реально та повністю працює** лише у двох окремих скриптах (`ea_fisheye_viewer.js`
> та `ea_fisheye_center.js`), які будують HTML-переглядач на базі **растрового PNG**.  
> Ці переглядачі **не мають** тултипів, редагування приміток та навігації drill-down —
> тільки чистий візуальний ефект лінзи. Цього достатньо для демонстрації можливостей технології.  
> Fisheye у SVG-файлі (v38) реалізований, але є дослідницьким прототипом —
> потребує доопрацювання для стабільної роботи на всіх конфігураціях діаграм.

Два скрипти-переглядачі. Fisheye попіксельний (алгоритм Фёрнаса, Xerox PARC 1986)
і застосовується до **растрового PNG** — це дає класичний рівномірний ефект
без прив'язки до структури SVG-елементів.

**Спільне для обох скриптів:**
- PNG діаграми: спершу пробуємо `Project.ExportDiagramToImage`, якщо недоступний —
  зберегти вручну через `File → Save Image → PNG` під ім'ям `ИмяДиаграммы_ID.png`
  у папку моделі, потім запустити скрипт ще раз (підхопить автоматично)
- PNG вбудовується в HTML як base64 → один автономний `.html`-файл
- Запис через `ADODB.Stream` (`Charset="UTF-8"`) — `CreateTextFile` не підтримує Unicode

**Чого немає в цих переглядачах (це PNG-перегляд, без SVG-функцій):**
- Тултипи з Notes елементів
- Редагування та збереження приміток читача
- Навігація drill-down/drill-up між діаграмами

### 3.1 `ea_fisheye_viewer.js` — лупа за мишею

**Концепція:** лінза слідує за курсором по нерухомій діаграмі.

**Що бачить користувач:**
- Діаграма на темному фоні, вписана в екран
- При наведенні миші: синє пунктирне кільце навколо курсору, вміст усередині збільшується
- За межами кільця — діаграма без змін

| Дія | Результат |
|---|---|
| Навести миш | Лінза слідує за курсором |
| Кнопка 👁 вкл/викл | Увімкнути/вимкнути лінзу |
| Слайдер «Сила D» | Інтенсивність збільшення (1–15) |
| Слайдер «Радіус R» | Розмір зони лінзи |
| Зум +/−/fit | Масштаб всього перегляду |
| `Пробіл` | Вкл/викл лінзу |
| `+` / `−` | Зум |
| `[` / `]` | Зменш/збільш радіус |
| `0` або `f` | Fit (вписати в екран) |

---

### 3.2 `ea_fisheye_center.js` — фіксована лінза, панорамування

**Концепція:** лінза нерухома в центрі екрану — ти «підтягуєш» потрібну ділянку
діаграми під неї. Оригінальний формат Focus+Context (Xerox PARC, 1986).

**Що бачить користувач:**
- Діаграма заповнює весь екран, спочатку вписана у вікно
- В центрі — синє пунктирне кільце з перехрестям (межа лінзи, завжди нерухома)
- Вміст усередині кільця збільшений, до країв стискається
- Перетягування мишею = панорамування діаграми під нерухомим збільшувачем

| Дія | Результат |
|---|---|
| Перетягнути мишею | Підтягнути потрібну ділянку до центру лінзи |
| Прокрутка колеса | Зум (в точку під курсором) |
| Кнопка 👁 вкл/викл | Увімкнути/вимкнути лінзу |
| Слайдери D, R | Параметри лінзи |
| Зум +/−/fit | Масштаб |
| `Пробіл` | Вкл/викл лінзу |
| `[` / `]` | Радіус лінзи |
| Стрілки ← → ↑ ↓ | Дрібний пан |
| `0` / `f` | Fit |

**Продуктивність:** поза лінзою — `ctx.drawImage()` (GPU, миттєво).
Піксельна обробка JS — тільки всередині кола R (~125K пікселів при R=200, ≈5–15ms/кадр).

---

### 3.3 Fisheye у SVG-файлі (v38) — статус: дослідницький прототип

Вбудований у головний SVG разом зі всіма функціями (тултипи, примітки, навігація).

**Технічна реалізація:**
- Елементи трансформуються через `translate(fx,fy) scale(s) translate(-cx,-cy)` на групі `<g>`
- Коннектори перераховуються покоординатно з `data-orig-d` (Liang–Barsky)
- Координати миші / diagScale → прив'язка в системі координат сцени

**Відмінність від растрового fisheye:** елементи «виростають» як блоки, а простір
між ними не деформується. Це технічно коректно, але відрізняється від класичного
попіксельного ефекту. Напрямок подальшого доопрацювання.

## 4. Служебные скрипты правки модели

### 4.1 `EA_add_composite_to_parent.js`

**Для кого:** администратор/разработчик модели EA.

**Что делает:** по всем элементам выделенной в браузере папки проверяет, есть ли связь **Composite** (Aggregation/Strong/DestIsAggregate=2) от элемента к элементу из его `ParentID`. Если нет — создаёт её `INSERT INTO t_connector`.

**Как запускать:**
1. В браузере EA выделить нужный пакет (папку)
2. `Specialize → Scripting → Run`
3. Результат — в диалоге: «Создано: N | Уже было: M | Пропущено: K»

**Что пропускает:**
- `ParentID = 0` / NULL
- Self-link (элемент указывает сам на себя)
- «Висячий» `ParentID` (объекта-родителя нет в модели)

**Идемпотентность:** повторный запуск ничего не дублирует — существующие связи проверяются из БД.

## 5. Инструмент регрессионной проверки

### `verify_direct.py`

**Для кого:** разработчик скрипта-экспортёра.

**Запуск:**
```bash
python3 verify_direct.py <model.xea> "<имя диаграммы>"
```

**Что делает:** читает модель напрямую (Python sqlite3), классифицирует каждый коннектор диаграммы на три типа:
- `DIRECT` — прямая линия (нет Path, нет TREE)
- `ORTHO-Path` — ортогональная с waypoints
- `TREE` — древовидный стиль

Для **DIRECT**-линий вычисляет конечные точки по правилу §5.2 (обратная формула + клип Liang–Barsky) и печатает их в EA-координатах. Сверяй с нативным экспортом EA для регрессионного контроля.

**Пример вывода:**
```
Diagram: Yuriy Chernyavskiy (ID 10015)

Conn 11607  Association    [DIRECT]
   SX/SY=-13/-14  EX/EY=-13/-18
   EA endpoints:  src=(254.0, -304.9)  tgt=(96.7, -206.0)

Conn 11731  Association    [ORTHO-Path]
   (left untouched -- waypoints/tree branch)
```
